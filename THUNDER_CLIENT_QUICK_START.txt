⚠️ THUNDER CLIENT SETUP
═══════════════════════════════════════════════════════════════════════════════

2. Select "Bearer Token"
3. Paste your token

OR manually add header:
1. Click "Headers" tab
2. Add: Authorization: Bearer <YOUR_TOKEN>

🔍 COMMON ISSUES & FIXES
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "Not authorized, no token provided"
FIX: Make sure Authorization header is set correctly
     Header: Authorization
     Value: Bearer YOUR_TOKEN

PROBLEM: "Invalid or expired token"
FIX: Token expired (>30 days)
     Solution: Login again (Step 3)

PROBLEM: "User role 'student' is not authorized"
FIX: Using student token for teacher action
     Solution: Use teacher token instead

PROBLEM: "Course not found"
FIX: Invalid course ID in URL
     Solution: Get ID from Step 5 or 6 response

PROBLEM: "You are already enrolled in this course"
FIX: Already enrolled in this course
     Solution: Can't enroll in same course twice

PROBLEM: "Cloud agent error"
FIX: Using Postman cloud (not applicable to Thunder Client)
     Thunder Client works fine locally
